# 00_Project_Brief｜项目立项说明

## 1. 项目名称

关系图谱

## 2. 项目类型

PWA 本地关系图谱工具。

## 3. 项目背景

用户希望拥有一个可以随时打开的关系记录工具，用于记录自己与身边人物的关系、互动事件、关系状态与备注。

由于用户当前使用 Windows 电脑，且没有 Mac / Xcode 环境，不适合直接开发原生 iOS App。因此项目采用 PWA 技术路线：

- Windows 可开发
- 不需要 Apple 开发者账号
- 不需要上架 App Store
- 可通过链接分享给朋友
- 朋友可添加到手机主屏幕
- 数据保存在各自设备本地

## 4. 项目目标

开发一款可添加到手机主屏幕使用的 PWA 应用，支持：

1. 添加人物。
2. 编辑人物资料。
3. 删除人物。
4. 记录“我”和人物的关系。
5. 记录互动事件。
6. 通过图谱展示人物关系。
7. 搜索和筛选人物。
8. 标签管理。
9. JSON 导入导出。
10. 离线缓存应用壳子。
11. 本地保存用户数据。

## 5. 核心边界

### 5.1 本项目要做

- PWA 前端应用。
- 本地数据库。
- 图谱可视化。
- 本地导入导出。
- 静态部署。
- 移动端适配。

### 5.2 本项目不做

- 后端服务器。
- 登录注册。
- 云同步。
- 多人协作。
- App Store 上架。
- 原生 iOS 安装包。
- 支付。
- 广告。
- 数据采集统计。
- 自动读取通讯录。
- 自动读取微信、短信或社交软件数据。

## 6. 用户使用流程

```text
用户打开链接
↓
进入关系图谱 PWA
↓
添加到主屏幕
↓
创建“我”的资料
↓
添加人物
↓
记录关系
↓
记录事件
↓
通过图谱查看关系网络
↓
定期导出 JSON 备份
```

## 7. 朋友使用流程

```text
你把部署链接发给朋友
↓
朋友用 Safari / Chrome 打开
↓
朋友添加到主屏幕
↓
朋友开始记录自己的关系图谱
↓
朋友的数据只存在朋友自己的设备本地
```

## 8. 成功标准

项目 V1.0 完成时，应满足：

- 能通过公网静态链接访问。
- 能添加到手机主屏幕。
- 能离线打开已缓存页面。
- 能新增、编辑、删除人物。
- 能记录人物关系。
- 能记录互动事件。
- 能展示基础关系图谱。
- 能搜索人物。
- 能导出和导入 JSON 数据。
- 不依赖后端服务器。
- 用户数据不上传。
