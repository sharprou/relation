# 02_Function_Spec｜功能详细说明

## 1. 初始化模块

### 1.1 自动创建“我”

首次进入应用时，系统检查本地数据库是否存在 `isSelf = true` 的人物。

如果不存在，创建默认人物：

| 字段 | 默认值 |
|---|---|
| name | 我 |
| relationType | 自己 |
| circle | 自我 |
| intimacy | 100 |
| trust | 100 |
| importance | 5 |
| status | 自己 |
| emotionalTone | 正向 |
| isSelf | true |

### 1.2 规则

- “我”节点只能存在一个。
- “我”不可删除。
- 如果导入数据后没有“我”，系统自动补建。
- 如果导入数据后有多个“我”，保留第一个，其余转为普通人物或提示修复。

## 2. 人物管理

### 2.1 新增人物

入口：

- 图谱页右上角新增按钮。
- 人物页右上角新增按钮。
- 空状态“添加第一个人物”。

字段：

| 字段 | 类型 | 必填 | 默认值 |
|---|---|---|---|
| 姓名 | 文本 | 是 | 无 |
| 昵称 | 文本 | 否 | 空 |
| 关系类型 | 枚举 | 是 | 普通认识 |
| 圈层 | 文本/枚举 | 否 | 未分类 |
| 亲密度 | 数值 | 否 | 50 |
| 信任度 | 数值 | 否 | 50 |
| 重要程度 | 数值 | 否 | 3 |
| 关系状态 | 枚举 | 否 | 正常 |
| 情绪倾向 | 枚举 | 否 | 中性 |
| 标签 | 多选 | 否 | 空 |
| 联系方式 | 文本 | 否 | 空 |
| 生日 | 日期 | 否 | 空 |
| 认识日期 | 日期 | 否 | 空 |
| 备注 | 长文本 | 否 | 空 |

保存后：

1. 创建 Person。
2. 创建 Relationship。
3. 返回人物列表或图谱页。
4. 图谱刷新。

### 2.2 编辑人物

可修改人物全部字段。

规则：

- 姓名不可为空。
- 修改亲密度时同步关系。
- 修改信任度时同步关系。
- 修改关系类型时同步关系。
- 修改状态时同步关系。
- 保存后更新 `updatedAt`。

### 2.3 删除人物

删除前弹窗确认：

```text
删除后将同时删除该人物相关关系与事件记录，且无法恢复。是否继续？
```

删除范围：

- Person
- 相关 Relationship
- 相关 InteractionEvent

“我”不可删除。

## 5. 图谱模块

### 5.1 展示规则

- “我”在中心。
- 其他人物围绕中心分布。
- 关系线连接“我”和人物。
- 节点颜色按圈层区分。
- 线条粗细按亲密度区分。
- 断联人物可降低透明度。
- 冲突关系可使用虚线或警示色。

### 5.2 交互规则

| 操作 | 结果 |
|---|---|
| 点击人物节点 | 进入人物详情 |
| 点击“我”节点 | 进入我的资料 |
| 拖拽画布 | 平移 |
| 缩放手势 | 放大/缩小 |
| 点击重置 | 回到默认视图 |
| 点击筛选 | 打开筛选面板 |

### 5.3 空状态

无人物时显示：

```text
还没有添加人物，先记录第一个人吧。
```

- 更新 Person.intimacy

- 更新 lastInteractionAt

数值范围限制为 0-100。

## 9. 备份恢复

### 9.1 导出

导出 JSON，包含：

- persons
- relationships
- events
- tags
- settings
- exportDate
- appVersion

文件名：

```text
relationship_graph_backup_yyyyMMdd_HHmm.json
```

### 9.2 导入

MVP 使用覆盖导入：

```text
清空当前数据 → 写入导入数据 → 检查“我”节点
```

导入前提示：

```text
导入数据可能覆盖当前数据，建议先导出当前数据作为备份。是否继续？
```
