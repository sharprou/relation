# 03_UI_UX_Spec｜UI 与交互说明

## 1. 设计目标

关系图谱 PWA 应具备轻量、清晰、私密、适合长期记录的视觉风格。

关键词：

- 私密
- 柔和
- 清晰
- 手账感
- 图谱感
- 不压迫
- 移动端友好

## 2. 整体布局

采用底部导航栏，包含 4 个主入口：

```text
图谱
人物
事件
设置
```

## 3. 页面结构

### 3.1 图谱页

内容：

- 顶部标题
- 筛选入口
- 重置视图按钮
- 关系图谱画布
- 新增人物浮动按钮

图谱交互：

- 点击节点进入详情
- 拖拽画布
- 缩放画布
- 筛选图谱
- 重置视图

### 3.2 人物页

内容：

- 搜索框
- 筛选按钮
- 人物卡片列表
- 新增按钮

人物卡片展示：

- 头像或首字母
- 姓名
- 关系类型
- 圈层
- 亲密度
- 标签

### 3.3 人物详情页

页面区块：

1. 基础信息区
2. 关系状态区
3. 标签区
4. 备注区
5. 事件时间线
6. 操作区

### 3.4 事件页

内容：

- 时间倒序事件列表
- 按人物筛选
- 按事件类型筛选
- 新增事件按钮

### 3.5 设置页

内容：

- 我的资料
- 标签管理
- 关系类型管理
- 数据导出
- 数据导入
- 清空本地数据
- 关于应用

## 4. 视觉规则

### 4.1 色彩建议

| 类型 | 色彩方向 |
|---|---|
| 背景 | 米白、浅灰、淡蓝 |
| 朋友 | 柔和绿色 |
| 家人 | 柔和粉色 |
| 工作 | 蓝色 |
| 学校 | 青色 |
| 网友 | 紫色 |
| 冲突 | 红橙色 |
| 断联 | 灰色 |

### 4.2 卡片样式

- 圆角 16px。
- 轻阴影。
- 信息层级清楚。
- 移动端单列布局。
- 桌面端可双列布局。

### 4.3 标签样式

- 胶囊形。
- 低饱和背景色。
- 小字号。
- 支持换行。

## 5. 图谱视觉

### 5.1 节点

节点包含：

- 圆形头像或首字母。
- 姓名。
- 外圈颜色。
- 重要人物高亮。

### 5.2 连线

| 数据 | 表现 |
|---|---|
| 亲密度高 | 线更粗 |
| 亲密度低 | 线更细 |
| 冲突 | 虚线 |
| 断联 | 灰色低透明 |
| 重要 | 高亮线 |

### 5.3 布局

MVP 使用圆形布局：

- “我”居中。
- 其他人物环绕分布。
- 亲密度越高，距离中心越近。
- 同圈层人物可尽量靠近。

## 6. 移动端适配

### 6.1 iPhone

- 底部导航高度适中。
- 表单输入不应被键盘遮挡。
- 图谱支持双指缩放。
- 卡片宽度占满屏幕。
- 详情页采用纵向滚动。

### 6.2 桌面端

- 人物列表可双列或三列。
- 图谱区域更大。
- 设置页居中显示。
- 支持鼠标滚轮缩放。

## 7. 空状态

### 7.1 人物为空

```text
你的人物关系图谱还是空的。
```

按钮：

```text
添加人物
```

### 7.2 事件为空

```text
暂无事件记录。
```

按钮：

```text
记录一件事
```

### 7.3 搜索无结果

```text
没有找到相关人物。
```

按钮：

```text
清空搜索
```

## 8. 危险操作确认

以下操作必须二次确认：

- 删除人物
- 删除事件
- 删除标签
- 清空全部数据
- 导入覆盖数据

## 9. PWA 安装引导

首次访问时，在设置页或首页展示引导：

### iPhone Safari

```text
点击 Safari 底部分享按钮 → 添加到主屏幕 → 完成。
```

### Android Chrome

```text
点击浏览器菜单 → 添加到主屏幕 / 安装应用。
```

## 10. 错误提示

| 场景 | 提示 |
|---|---|
| 姓名为空 | 请输入人物姓名 |
| 事件标题为空 | 请输入事件标题 |
| 导入失败 | 文件格式错误，无法导入 |
| 导出失败 | 导出失败，请稍后重试 |
| 数据库异常 | 本地数据读取失败，请尝试刷新页面 |

## 11. 体验原则

- 表单非必填项不要阻碍保存。
- 不要一开始展示过多字段。
- 详情页可展示完整信息，列表只展示核心信息。
- 图谱页重展示，编辑操作放到详情页。
- 移动端优先，桌面端增强。
