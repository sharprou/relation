## 项目总提示词

你是一个资深前端工程师，请协助开发一个名为「关系图谱」的 PWA 本地关系记录工具。

## 项目定位

这不是原生 iOS App，而是一个可以部署到静态网页平台，并添加到手机主屏幕使用的 PWA。

用户通过链接打开应用，添加到主屏幕后像软件一样使用。

## 技术栈

- React
- TypeScript
- Vite
- Tailwind CSS
- React Router
- Zustand
- Dexie.js
- IndexedDB
- React Flow / @xyflow/react
- vite-plugin-pwa

## 运行环境

- Windows 开发
- VS Code 编写
- Chrome / Edge 调试
- iPhone Safari 添加到主屏幕
- Android Chrome 添加到主屏幕

## 核心要求

1. 不接入后端服务器。
2. 不接入登录注册。
3. 不上传用户数据。
4. 不接入广告。
5. 不接入支付。
6. 不自动读取通讯录。
7. 不自动读取社交软件。
8. 所有人物、关系、事件、标签数据保存在用户设备本地 IndexedDB。
9. 支持 JSON 导出和导入。
10. 应用可添加到手机主屏幕。
11. 公网部署只用于分发静态前端文件，不保存业务数据。

## 核心功能

### 1. 人物管理

- 新增人物
- 编辑人物
- 删除人物
- 查看人物详情
- 人物搜索
- 人物筛选

### 2. 关系管理

- 自动创建“我”节点
- 新增人物时自动创建“我 → 人物”的关系
- 记录关系类型
- 记录关系状态
- 记录亲密度
- 记录信任度
- 记录情绪倾向

### 3. 图谱展示

- 使用 React Flow
- “我”节点居中
- 人物节点围绕展示
- 连线展示关系
- 节点颜色按圈层区分
- 连线粗细按亲密度区分
- 点击节点进入人物详情

### 4. 事件记录

- 新增事件
- 编辑事件
- 删除事件
- 人物详情中展示事件时间线
- 事件可影响亲密度和信任度

### 5. 标签系统

- 新增标签
- 编辑标签
- 删除标签
- 人物可多选标签
- 可按标签筛选

### 6. 数据备份

- 导出 JSON
- 导入 JSON
- 清空本地数据
- 导入前二次确认
- 清空前二次确认

### 7. PWA

- manifest
- icons
- service worker
- standalone display
- 添加到主屏幕说明
- 静态部署支持

## 开发原则

- 每完成一个阶段，项目必须可运行。
- 不要过度设计。
- 优先保证核心功能闭环。
- 图谱先使用简单圆形布局。
- 移动端优先。
- 所有危险操作必须二次确认。
- 代码结构清晰，方便维护。
- 每次完成后说明修改文件和测试方式。
