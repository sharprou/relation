# User_Guide：关系图谱 PWA 用户使用说明

## 如何开始

1. 打开应用链接。
2. 按设置页说明将应用添加到手机主屏幕。
3. 进入人物页，添加第一个人物。
4. 回到图谱页，查看“我”和人物之间的关系图谱。

首次打开时，应用会自动创建“我”节点。“我”节点用于作为关系图谱中心，不会作为普通人物删除。

## 如何记录人物

在人物页点击“新增人物”，填写人物资料后保存。

常用字段包括：

- 姓名：必填。
- 昵称：可选，用于记录别称。
- 关系类型：例如朋友、同事、家人、合作方。
- 圈层：例如工作、学校、家庭、兴趣圈。
- 亲密度：0-100，用于描述关系亲近程度。
- 信任度：0-100，用于描述信任程度。
- 重要程度：1-5，用于标记关系重要性。
- 状态：例如正常、亲近、疏远、观察、冲突。
- 情绪倾向：正向、中性、负向、复杂。
- 标签：用于后续搜索和筛选。
- 备注：自由记录补充信息。

保存人物后，应用会自动创建一条“我 -> 该人物”的关系，并在图谱页显示。

## 如何记录事件

在事件页点击“记录一件事”，选择关联人物并填写事件标题、类型、日期、情绪倾向和备注。

事件可以选择是否影响关系数值：

- 勾选“影响关系数值”后，可以填写亲密度变化和信任度变化。
- 保存后，人物和对应关系的亲密度 / 信任度会同步更新。
- 数值会限制在 0-100 范围内。

人物详情中会展示该人物相关事件时间线。

## 如何使用标签

在设置页可以新增、编辑和删除标签。人物表单中可以选择多个标签。

标签可用于：

- 人物页搜索
- 人物页筛选
- 图谱页筛选
- 快速标记人物特征

删除标签时，应用会二次确认；确认后，该标签会从所有人物资料中同步移除。

## 如何备份数据

建议定期导出 JSON 备份。

操作方式：

1. 打开设置页。
2. 点击“导出 JSON 备份”。
3. 保存下载的 `.json` 文件。

备份文件包含：

- persons
- relationships
- events
- tags
- settings
- exportDate
- appVersion

备份文件只保存在用户自己选择的位置，不会上传到服务器。

## 如何恢复数据

1. 打开设置页。
2. 点击“导入 JSON 备份”。
3. 选择之前导出的 `.json` 文件。
4. 确认覆盖导入。

导入是覆盖导入，会替换当前设备上的本地数据。建议导入前先导出一份当前数据备份。

如果选择了错误文件或格式不正确的 JSON，应用会提示失败，并且不会清空当前数据。

## 如何换设备

1. 在旧设备打开设置页。
2. 导出 JSON 备份。
3. 将 JSON 文件传到新设备。
4. 在新设备打开应用链接。
5. 添加到主屏幕。
6. 打开设置页并导入 JSON 备份。

导入后，人物、关系、事件、标签和图谱会恢复到新设备本地。

## 如何清空本地数据

在设置页点击“清空本地数据”。该操作会二次确认。

确认后，应用会删除当前设备中的人物、关系、事件、标签和设置，并重新创建“我”节点。

清空前建议先导出 JSON 备份。

## 数据安全提示

- 数据只保存在当前设备本地。
- 应用不接入后端服务。
- 应用不接入登录注册。
- 应用不上传用户数据。
- 应用不做云同步。
- 清理浏览器数据、删除站点数据或卸载 PWA 可能导致数据丢失。
- 卸载 PWA 或清理站点数据前，应先导出 JSON 备份。

## 常见问题

### 为什么换手机后看不到旧数据？

数据保存在旧设备本地，不会自动同步到新设备。请先在旧设备导出 JSON，再在新设备导入。

### 添加到主屏幕后，数据会上传吗？

不会。添加到主屏幕只是让应用像 App 一样打开，数据仍然保存在当前设备本地。

### 离线时可以使用吗？

首次联网打开后，PWA 会缓存基础应用壳子。之后基础页面可以离线打开，业务数据仍从本机 IndexedDB 读取。若浏览器清理了站点数据，数据可能丢失。
